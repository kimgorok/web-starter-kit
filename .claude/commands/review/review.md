# Claude 명령어: Code Review

현재 변경사항에 대한 종합적인 코드 리뷰를 진행합니다.

## 사용법

```
/review
```

## 프로세스

1. 현재 변경사항 분석 (git diff)
2. 변경된 파일 목록 확인
3. 코드 품질, 버그, 보안, 성능, 스타일 검토
4. 개선 제안 작성
5. 종합 리뷰 결과 정리

## 리뷰 기준

### 🐛 버그 & 논리 에러
- 잠재적 런타임 에러
- 예외 처리 누락
- 타입 에러
- 논리적 결함

### 🔒 보안 & 접근성
- 민감한 데이터 노출
- 인증/권한 문제
- XSS, SQL Injection 취약점
- 접근성 고려사항

### ⚡ 성능 & 메모리
- 불필요한 반복
- 메모리 누수 위험
- 비효율적 알고리즘
- 번들 크기 영향

### 🎨 코드 스타일 & 컨벤션
- 명명 규칙 준수
- 코드 일관성
- 주석 및 문서화
- 가독성 개선

### 📋 아키텍처 & 설계
- 디자인 패턴 준수
- 코드 재사용성
- 컴포넌트 책임 분리
- 의존성 관리

### 🧪 테스트 & 검증
- 테스트 커버리지
- 엣지 케이스 처리
- 수동 테스트 필요 여부

## 리뷰 체크리스트

- [ ] **기능**: 의도한 대로 동작하는가?
- [ ] **버그**: 잠재적 에러가 없는가?
- [ ] **보안**: 보안 취약점이 없는가?
- [ ] **성능**: 성능상 문제가 없는가?
- [ ] **스타일**: 코드 컨벤션을 따르는가?
- [ ] **테스트**: 충분히 테스트되었는가?
- [ ] **문서화**: 필요한 부분이 문서화되었는가?
- [ ] **접근성**: 접근성 기준을 고려했는가?

## 리뷰 결과 형식

### 📊 리뷰 요약
- 변경 파일 수
- 주요 변경 사항
- 전체 평가

### ✅ 좋은 점
- 잘 구현된 부분
- 개선된 부분

### ⚠️ 개선 제안
- 우선순위: High / Medium / Low
- 구체적인 개선 방법
- 코드 예시 (필요시)

### 📌 추가 검토 필요
- 테스트 확인 필요
- 성능 검증 필요
- 보안 감시 필요

## 참고사항

- 정보성 리뷰: 코드 품질 개선에 초점
- 건설적 피드백: 부정적 표현 지양
- 자동화 도구 활용: ESLint, TypeScript 검사
- 팀 컨벤션 준수: 프로젝트 규칙 우선
- 문제 재현: 로컬에서 테스트 가능한 경우 검증
